Understand any repository

$ blink context

Identity, measured statistics, and the project's most significant areas — ranked by symbol density, so the substance surfaces first.

Search with context

$ blink query "where are the API routes"

Find relationships, not just text. Blink drops the question words and ranks the areas, files, and symbols that actually match — over the local model, with no LLM in the loop.

Explain any file

$ blink explain src/auth/login.rs

Deterministic, real-signal understanding: the file's own doc, the symbols it declares, what it imports, and what imports it.

  • No language model, no guessed "responsibilities"
  • Every line read from the file or the index

Built for the AI era

AI tools are powerful.
Context makes them reliable.

AI can generate code in seconds — but it stumbles on the same things every developer does in an unfamiliar repository: the structure, the conventions, the dependencies, the blast radius of a change. Blink is the structured, verifiable understanding that reliable tooling is built on.

Blink ships no AI and calls no model. It's the context layer that AI tooling lacks — local, deterministic, and yours.

Project configuration

Tune how Blink
understands your project.

A single signature .bnk file (or blink.toml) configures detection, indexing, commands, and the context engine. No config? Blink works on sensible defaults.

.bnk
[project]
name = "my-app"

[context]
enabled = true
include = ["src", "docs"]

[commands]
dev  = "npm run dev"
test = "npm test"
Principles

Understanding you can trust.

01

Local-first

Runs entirely on your machine. Nothing is uploaded, ever.

02

Offline & fast

No network by default. An incremental index keeps repeat runs instant.

03

Deterministic

Same bytes on disk, same graph. Reproducible by construction.

04

No fabricated output

Every count, symbol, and edge is measured or resolved — never invented.
